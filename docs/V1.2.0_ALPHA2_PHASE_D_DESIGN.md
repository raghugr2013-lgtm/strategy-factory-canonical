# Phase D — Adaptive Autonomous Portfolio & Master Bot Engine

**Status:** Design + implementation. Additive over Phase C.
**Ultimate goal:** Autonomous institutional-grade portfolio manager. The operator only supplies credentials, capital limits, and (optionally) approves production deploys — everything else runs continuously without intervention.

---

## 1. Architectural pivot

Before Phase D: strategies were first-class citizens. Ranking was `top-N by profit_factor`, Master Bots stored a static tier list.

After Phase D: **portfolios are first-class citizens**. A Master Bot is an adaptive portfolio of strategies across styles (trend, mean_reversion, breakout, momentum, session, volatility, news, recovery, scalping, swing) with continuously-rebalanced capital allocations. Individual strategies are components that get promoted, demoted, replaced, and retired autonomously.

## 2. Engines added (`backend/legacy/engines/portfolio/`)

| # | Engine | Responsibility |
|---|--------|---------------|
| 1 | `allocation.py` — **Portfolio Allocation Engine** (the "brain") | For every strategy in the current portfolio, produces one of {ACTIVATE, PAUSE, REDUCE, INCREASE, REPLACE, HOLD} based on market regime × confidence × correlation × drawdown × risk × exposure × capital × outcome learning. |
| 2 | `capital.py` — **Capital Allocation Engine** | Converts allocation decisions into % weights (must sum to ≤100%) with a mandatory cash reserve. Style-aware (Kelly-inspired risk-parity blend). |
| 3 | `health.py` — **Portfolio Health Engine** | Continuous monitor of correlation / drawdown / style concentration / pair concentration / exposure / diversity. Emits rebalance signals when any threshold is breached. |
| 4 | `promotion.py` — **Strategy Promotion Engine** | Autonomous pipeline: Research → Validated → Tier 3 → Tier 2 → Tier 1 → Production. Promotion criteria (per stage) are stateless functions over recent outcome events. |
| 5 | `retirement.py` — **Strategy Retirement Engine** | Detects degradation (PF trend, WR trend, confidence decay, prediction accuracy drop) using rolling windows and demotes / archives strategies. |
| 6 | `rebuilder.py` — **Self-Rebuilding Master Bot** | Composes 1–5 into a single "rebuild pass". Called by the orchestrator every N minutes. |
| 7 | `closed_learning.py` — **Continuous Closed Learning** | Records the *predicted → realised* delta for every autonomous decision, feeds it back into strategy confidence and outcome events, so the system learns from its own past decisions. |
| 8 | Dynamic Strategy Selector — **already delivered in Phase C** (`engines/intelligence/dynamic_selector.py`). Phase D wires it into the rebuilder so the Master Bot activates the right strategy for the current regime on every rebuild pass. |

Every engine is:
* **Pure / stateless** where possible — reads inputs, returns a `Decision` dataclass.
* **Explainable** — the `Decision` carries `reason` + `evidence`, and `_emit_decision()` writes one `outcome_events` row per meaningful autonomous action.
* **Fail-open** — missing inputs degrade to conservative behaviour (HOLD / no-op).

## 3. Data contracts

### `PortfolioMember`
```python
strategy_hash: str
style:         str          # from Phase C classifier
confidence:    float        # from Phase C classifier
allocation:    float        # 0..1 — current portfolio weight
status:        str          # active | paused | research
tier:          str          # tier_1 | tier_2 | tier_3 | research | production
backtest:      dict         # profit_factor, max_drawdown_pct, win_rate, total_trades
equity_curve:  list[float]  # optional — enables correlation math
recent_outcomes: list[dict] # last N outcome events for this strategy
```

### `PortfolioAction`
```python
strategy_hash: str
action:        Literal["ACTIVATE","PAUSE","REDUCE","INCREASE","REPLACE","HOLD"]
weight_delta:  float        # signed change in allocation
reason:        str
evidence:      dict
```

### `PortfolioState`
```python
master_bot_id: str
tier:          str
members:       list[PortfolioMember]
cash_reserve:  float        # 0..1 — fraction held out of the market
last_rebuild:  str          # ISO timestamp
health:        dict         # snapshot from Health Engine
```

## 4. Orchestrator integration

New task `self_rebuild` (workload=API_HOT, dependencies=`(ranking,)`, freshness SLA 15 min, PASSIVE by default → operator activates via env). Task adapter drives one full rebuild pass:

```
load portfolio state
  ↓
health check          → PortfolioHealthReport
  ↓
promotion evaluation  → list[promotions]
  ↓
retirement evaluation → list[retirements]
  ↓
allocation decisions  → list[PortfolioAction]     ← Portfolio Allocation Engine
  ↓
capital reweight       → dict[hash → weight]        ← Capital Allocation Engine
  ↓
dynamic selection     → ActivationDecision         ← Phase C Selector
  ↓
persist + emit outcome events
```

## 5. API surface

All under `/api/portfolio/*`, additive:

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET  | `/portfolio/health/{master_bot_id}` | user | Health Engine snapshot |
| POST | `/portfolio/allocate` | user | Run Allocation Engine on a supplied portfolio state |
| POST | `/portfolio/capital` | user | Run Capital Allocation Engine (converts actions → weights) |
| GET  | `/portfolio/promotion-candidates` | user | Strategies eligible for promotion |
| GET  | `/portfolio/retirement-candidates` | user | Strategies eligible for retirement |
| POST | `/portfolio/rebuild/{master_bot_id}` | **admin** | Full self-rebuild pass (idempotent) |
| GET  | `/portfolio/state/{master_bot_id}` | user | Current adaptive portfolio state |
| POST | `/portfolio/closed-learning/record` | user | Record a realised outcome for an autonomous decision |

| `PORTFOLIO_REBUILD_MAX_CHANGES` | `5` | Max concurrent portfolio changes per rebuild |

## 7. Explainability

Every autonomous action emits one `outcome_events` row via `intelligence.emit_decision` with:
* `decision_type` ∈ {`allocation_action`, `capital_reweight`, `health_check`, `promotion`, `retirement`, `master_bot_rebuild`, `closed_learning_feedback`}
* `strategy_hash` (when strategy-scoped)
* `evidence` — full input snapshot + engine derivation

Queryable via `/api/learning/events?decision_type=…`.
