# Phase G — Market Intelligence

**Status:** DESIGN ONLY. Awaiting operator sign-off before implementation (same protocol as Phase F).

**Additive · backward compatible · instant-rollback via feature flag.**

---

## 1. Objective

Phase A–F taught the factory to learn **from strategy performance**. Phase G teaches it to learn **from the market itself**, so it can adapt *before* strategies begin failing rather than after.

The Adaptive Trading Brain currently consumes:
* strategy classification confidence
* strategy backtest / recent PF / drawdown / prediction accuracy
* portfolio correlation / diversification
* regime snapshot from Phase C

After Phase G it will additionally consume:
* rolling **MarketState ledger** — trend duration, volatility expansion/contraction, breakout success rate, reversal strength, session behaviour, liquidity, cross-pair correlation
* **Structural Change signals** — volatility regime shifts, trend duration drift, breakout degradation, noise increase, correlation breakdown
* **Market Intelligence scores** — Market Confidence, Style Confidence, Regime Confidence, Opportunity Score, Risk Environment Score

These signals plug into the existing `BrainSignals` struct via a strictly-additive path so `PORTFOLIO_POLICY=phase_d` still runs byte-identically.

---

## 2. Non-goals

* **NOT** more strategy generation.
* **NOT** ML training. The change-point detectors and scorers are heuristic + rolling-statistics in this phase; ML/HMM/Bayesian is roadmap (Phase G.2 or beyond).
* **NOT** a UI change.
* **NOT** a live-execution feed. That is Phase H.
* **NOT** meta-learning of the brain's own weights. That is Phase I.

## 4. Data contracts

### `MarketSnapshot`
One observation, one timestamp:
```python
pair:            str
timeframe:       str
ts:              str
close:           float
range_pct:       float
volatility:      float                    # rolling stdev of returns
volume:          Optional[float]          # if available
session:         str                      # asian | london | ny | overlap | quiet
regime:          str                      # from Phase C
trend_score:     float                    # −1..1
```

### `MarketState`
Rolling aggregation, one row per (pair, timeframe, window):
```python
pair:                        str
timeframe:                   str
window:                      str          # e.g. "24h" | "7d" | "30d"
ts:                          str
trend_duration_bars:         float        # avg length of unbroken trend legs
trend_persistence_score:     float        # 0..1
volatility_mean:             float
volatility_expansion_ratio:  float        # recent σ / long-window σ
breakout_attempts:           int
breakout_success_rate:       float        # 0..1
reversal_strength_avg:       float        # 0..1 (depth × velocity)
noise_ratio:                 float        # false signals / total signals
session_pnl_bias:            Dict[str, float]  # per-session directional bias
liquidity_band:              str
avg_correlation_to_universe: Optional[float]
style_performance:           Dict[str, float]  # style → rolling PF proxy
health_score:                float        # 0..1 composite
```

### `StructuralChange`
```python
pair:               str
timeframe:          str
change_type:        str   # volatility_regime_shift | trend_duration_drift
                          # | breakout_degradation | noise_increase
                          # | liquidity_drop | correlation_breakdown
                          # | regime_transition_confirmed
severity:           float # 0..1
detected_at:        str
window_before:      str
window_after:       str
delta_metric:       Dict[str, float]      # what moved and by how much
evidence:           Dict[str, Any]
```

### `MarketIntelligence`
The consumable payload for the Trading Brain:
```python
pair:                 str
timeframe:            str
ts:                   str
market_confidence:    float        # 0..1 — overall clarity of the market
style_confidence:     Dict[str, float]  # per-style: mean_reversion, trend, …
regime_confidence:    float        # 0..1 — same regime as Phase C but validated by structure
opportunity_score:    float        # 0..1 — high when signals align
risk_environment:     float        # 0..1 — high when risk environment is benign
active_structural_changes: List[StructuralChange]
sources:              Dict[str, Any]     # per-observer sub-scores for explainability
```

## 5. Persistence

New Mongo collections (additive, indexed):

| Collection | Purpose | Key fields |
|---|---|---|
| `market_snapshots` | Raw observations. TTL 30 days. | `(pair, timeframe, ts)` |
| `market_states` | Rolling aggregations. | `(pair, timeframe, window, ts)` |
| `structural_changes` | Detected change points. | `(pair, timeframe, change_type, detected_at)` |
| `market_intelligence` | Latest MarketIntelligence per (pair, timeframe). Upsert. | `(pair, timeframe)` |

All writes go through `ledger.py` — no other module writes to these collections. This keeps the source of truth single and easy to reason about.

### Indexes
* `market_snapshots`: `(pair, timeframe, ts DESC)` + TTL on `ts`
* `market_states`: `(pair, timeframe, window, ts DESC)`
* `structural_changes`: `(pair, timeframe, detected_at DESC)` + `(change_type, detected_at DESC)`
* `market_intelligence`: `(pair, timeframe)` unique

## 6. Observers (§7 of operator brief)

Each observer is a pure function `f(snapshots: List[MarketSnapshot], config) -> ObserverResult`.

| Observer | What it produces |
|---|---|
| `trend_duration` | avg + std of consecutive same-sign moves; persistence score |
| `volatility_dynamics` | short_σ / long_σ ratio; expansion/contraction flag |
| `breakout_quality` | success rate of last N Donchian/high-low breakouts |
| `reversal_strength` | depth × velocity of last N reversals; composite 0..1 |
| `session_stats` | per-session mean return, hit rate, dominant regime |
| `liquidity_estimator` | proxy from range/spread + session; low/medium/high |
| `correlation_matrix` | rolling pairwise correlation vs universe |
| `style_performance` | rolling proxy PF per style class (derives from strategy backtest metadata + regime tags) |

Every observer:
* is deterministic
* takes a config object read from env (windows, thresholds)
* returns a dataclass with `score`, `evidence`, `ts`
* is unit-testable without any Mongo

## 7. Structural change detection

`change_detection.py` implements three heuristic detectors (no ML yet — Q3 direction from Phase F applies here too):

* **CUSUM-lite** on volatility_mean — detects regime shifts.
* **Running-mean deviation** on trend_duration_bars — detects trend-duration drift.
* **Rolling ratio drop** on breakout_success_rate — detects breakout degradation.
* **Correlation breakdown** = |corr_current − corr_baseline| > threshold.
* **Noise increase** = false_signals / total_signals rising above baseline + 2σ.

Method label on every emitted `StructuralChange` follows the Phase F convention (`method="heuristic_*"`) so a future ML replacement can plug in without breaking the contract.

## 8. Market Intelligence aggregator

`intelligence.py` runs on demand or via orchestrator task. Steps:

1. Read the latest `MarketState` for (pair, timeframe, "24h").
2. Read active `structural_changes` (severity ≥ threshold, detected_at within 24 h).
3. Compute:
   * `market_confidence` = weighted composite of `trend_persistence_score`, `breakout_success_rate`, `1 − noise_ratio`, `regime_confidence` from Phase C.
   * `style_confidence` per style from `style_performance` + `regime_confidence` × `regime_style_fit`.
   * `regime_confidence` — inherited from Phase C, discounted by active `structural_changes`.
   * `opportunity_score` = `market_confidence × best_style_confidence × liquidity_factor`.
   * `risk_environment` = `1 − (volatility_expansion_ratio_penalty + noise_penalty + change_severity_penalty)`.
4. Upsert into `market_intelligence` collection.
5. Emit `outcome_events` row with `decision_type="market_intelligence_refresh"` and full evidence.

## 9. Trading Brain integration

`brain_bridge.py` exposes:
```python
async def load_market_intelligence(pair, timeframe) -> Optional[MarketIntelligence]
```

`engines/brain/signals.py` gets an additive block:
```python
mi = await load_market_intelligence(pair, timeframe) if bcfg.brain_uses_market_intelligence() else None
signals.market_confidence   = mi.market_confidence   if mi else None
signals.style_confidence    = mi.style_confidence    if mi else {}
signals.opportunity_score   = mi.opportunity_score   if mi else None
signals.risk_environment    = mi.risk_environment    if mi else None
signals.structural_changes  = [c.to_dict() for c in mi.active_structural_changes] if mi else []
```

`BrainSignals` gets 5 new **optional** fields, all with safe defaults. Any tick without Phase G data behaves identically to today.

`engines/brain/scorer.py` gets three additive scoring terms controlled by env weights (default 0.0 → **no behaviour change until operator opts in**):
* `BRAIN_W_MARKET_CONFIDENCE` (default 0.0)
* `BRAIN_W_STYLE_CONFIDENCE` (default 0.0)
* `BRAIN_W_OPPORTUNITY`      (default 0.0)

When the operator sets these ≥ 0 the brain's `score_now` starts consuming market intelligence.

Additionally, `policy.py` gets one new hook (behind a flag):
* If `risk_environment < BRAIN_MARKET_RISK_PAUSE_THRESHOLD`, force `PAUSE` on strategies whose `style_confidence[member.style] < BRAIN_MARKET_STYLE_MIN_CONFIDENCE`.

Default thresholds are set so the hook is dormant.

## 10. Orchestrator integration

New task `market_intelligence_refresh`:

* `WORKLOAD_CLASS = API_HOT`
* `DEPENDS_ON = ("market_data_topup",)`
* `MIN_INTERVAL_S = 300` (5 min)
* `PRIORITY_BASE = 65`
* `PASSIVE = False` — active by default because it's cheap (no LLM), no external calls.
* Reads latest snapshots per configured universe, runs observers → change detection → aggregator, upserts `market_intelligence`.

The `self_rebuild` task (Phase D) already consults Phase C regime; after Phase G it also consults the latest `market_intelligence` when `PORTFOLIO_POLICY=brain` AND market weights > 0.

## 12. Explainability

Every autonomous market-driven decision writes one `outcome_events` row via the existing `intelligence.emit_decision` helper:

| decision_type | Emitted by | Metrics | Evidence |
|---|---|---|---|
| `market_state_refresh` | ledger on every append | window, health_score, key scalars | observer sub-scores |
| `structural_change_detected` | change_detection | change_type, severity | before/after windows + delta_metric |
| `market_intelligence_refresh` | aggregator on every upsert | market_confidence, opportunity, risk_env | per-observer contributions + active changes |
| `brain_market_influence` | scorer whenever market weight > 0 | strategy_hash, market_confidence, style_confidence[style] | before/after score_now, weight applied |
| `brain_market_risk_pause` | policy when market-risk force-pause fires | strategy_hash, risk_environment | style_confidence, threshold |

Dashboard question "**Why did the brain increase this strategy?**" is answered by chaining:
1. `brain_decision` (Phase F) — shows the 9 base signal components.
2. `brain_market_influence` (Phase G) — shows the market-intelligence bump.
3. `market_intelligence_refresh` — shows which observers drove the score.
4. `structural_change_detected` — shows the market events that shaped the observers.

Full traceability from a portfolio weight delta back to a raw price observation.

## 14. Extensibility contracts (Phase H / I / Self-Evaluation)

Explicit hooks are pre-built now so future phases plug in without refactoring:

* **Phase H — Live Execution Intelligence**
  * Observers `liquidity_estimator` and `session_stats` read from a pluggable `MarketDataSource` protocol. `MarketDataSource.get_snapshots(pair, timeframe, since)` will get a `LiveCtraderDataSource` implementation in Phase H that streams realised fills + slippage.
  * `MarketState.session_pnl_bias` will accept live-PnL updates from a Phase H reporter.

* **Phase I — Meta-Learning**
  * Every scoring weight (both `BRAIN_W_*` and `MI_W_*`) is loaded via a `WeightProvider` protocol. Default reads env; Phase I's `MetaLearnedWeightProvider` reads from Mongo `learned_weights` collection and adjusts based on closed-learning deltas.
  * `outcome_events` rows tagged with `decision_type` prefixes `market_*` and `brain_market_*` are the training data.

* **Future Self-Evaluation Layer**
  * `MarketIntelligence.sources` retains per-observer contributions, so a daily "Factory Performance Review" job can:
    * measure prediction accuracy of `opportunity_score` vs realised portfolio PnL
    * compute AI-spend efficiency (cost per point of `market_confidence` gained)
    * flag observers whose signal has decayed
  * A `daily_review` collection is pre-reserved (no writes yet).

## 16. Open architecture questions — for operator review

Before I implement, please confirm:

* **Q1 · Universe.** Default `MI_UNIVERSE=EURUSD,GBPUSD,USDJPY,XAUUSD` and `MI_TIMEFRAMES=H1,H4,D1`. Confirm or supply your production universe. Wider universes multiply Mongo storage roughly linearly.
* **Q2 · Ingestion of snapshots.** Options:
  * **(a)** New micro-task `market_snapshot_ingest` runs every minute on the orchestrator and writes to `market_snapshots`.
  * **(b)** Piggyback on the existing `market_data_topup` task — every top-up also emits a MarketSnapshot as a side effect.
  * Recommendation: **(b)** — one code path for market I/O.
* **Q3 · Structural change persistence.** Every detected change writes an `outcome_events` row *and* a dedicated `structural_changes` row (queryable + timeline-friendly). Confirm or vote for events-only.
* **Q4 · Brain default weights.** All `BRAIN_W_MARKET_*` default to `0.0` so activating Phase G is a two-step operator opt-in (`BRAIN_USES_MARKET_INTELLIGENCE=true` + set weights). Confirm this two-step cautious activation is the right default, or should the master switch alone (`BRAIN_USES_MARKET_INTELLIGENCE=true`) auto-set the weights to a sensible starting mix (e.g. 0.05/0.05/0.05)?
* **Q5 · Market-driven force-pause.** The `BRAIN_MARKET_RISK_PAUSE_THRESHOLD` hook is off by default. Do you want it *on by default* once the master switch is flipped (risk-first stance) or *off by default* (operator explicitly opts in when comfortable)?
* **Q6 · Style-performance observer scope.** Should `style_performance` compute rolling PF proxy from:
  * **(a)** live outcome events only (drift-free, but slow to warm up)
  * **(b)** live outcome events plus recent backtests (fast warm-up, some evidence bleed)
  * Recommendation: **(a)** — cleaner separation between market truth and strategy noise.

## 17. Success criteria for Phase G sign-off

* `MI_ENABLED=false` behaviour byte-identical to Phase F (regression gate).
* `MI_ENABLED=true`, `BRAIN_USES_MARKET_INTELLIGENCE=false` → ledger + observers + change detection run continuously and produce `outcome_events`, but the brain does NOT consume them (drift-free monitoring).
* `MI_ENABLED=true`, `BRAIN_USES_MARKET_INTELLIGENCE=true`, market weights > 0 → brain score_now measurably shifts when a structural change is injected on synthetic data (test-provable).
* Every autonomous market-driven decision has a full evidence chain in `outcome_events` from raw snapshot → observer → aggregator → brain influence → portfolio weight delta.
* Regression suite grows from 204 to ~234 tests, all passing.
